# How-Software-Is-Born

***The stories behind the code***

Every application has two versions. The one users see. And the one that exists only inside the developer's mind. This repository is about the second one.

Most repositories show the final result. This one doesn't. This repository is about everything that happens before the first line of code is written. Every project starts with an idea. Then come the questions. The research. The wrong assumptions. The refactorings. The compromises. Finally ... the code. This repository documents that journey. Not because there is only one correct solution.

***Why this repository exists?***

I learn best by building. Whenever I study a new technology, I create a personal project. Not to showcase the technology. But to understand how it changes the way I think. The project becomes my laboratory. The code becomes the result. The real work happens long before that.

***What you will find here?***

This is not a tutorial. It is not a collection of code snippets. It is a collection of engineering decisions. Each lesson starts with a problem. Then explores different ideas. Explains why one solution was chosen. And discusses the trade-offs behind that decision. Sometimes the final answer changes. That is part of engineering too.

***The first project!***

The first project documented here is Viora. A simple personal application that originally started as an experiment to learn SwiftData. It slowly became something much more interesting: a journey about modeling ideas before modeling data.
